## Markdown简介及常用命令

### 简介

Markdown语言是一种轻量级标记语言，已经被许多网站广泛用于撰写帮助文档或论坛消息，比较常见的是GitHub中许多项目库的README.md文件。使用Markdown进行文档编写具有以下几项优势：

- 操作简单，双手无需离开键盘即可完成常见的格式配置
- 纯文本内容，可以使用任何文本编辑软件
- 搭配Git可以很好**管理文档版本**，不会造成版本混乱（与Word相比）
- 体积小，易于保存和分享
- 可以**跨平台使用**，在Windows/Linux/MacOS上均可使用（与MS Office相比）
- ……

考虑到上述优势，学习使用Markdown将大大方便我们的学习生活。与其他工具的配套使用能够更大的发挥其作用。

#### Markdown标题

##### 使用`=`和`-`标记一级标题、二级标题

源代码如下：

```markdown
一级标题
====

二级标题
----
```

##### 使用`#`标记1-6级标题

一级标题对应一个`#`，二级标题对应两个`#`，以此类推，源代码如下：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

#### Markdown段落格式

##### 分隔线

在同一行中用三个以上的`*`、`-`、`_`，行内不能有其他内容。可以在`*`或`-`中间插入空格，源代码如下：

```markdown
***

* * *

*****

- - -

----------
```

任意一行的显示效果如下：

---

##### 脚注

Markdown脚注源代码如下：

```markdown
脚注格式示例 [^1]。

[^1]: 示例文字
```

效果如下：

脚注格式示例 [^1]。

[^1]: 示例文字

#### Markdown列表

##### 无序列表

使用`*`、`+`、`-`作为列表标记，标记后空一格再填写内容，源代码如下：

```markdown
* 第一项
* 第二项
* 第三项

+ 第一项
+ 第二项
+ 第三项

- 第一项
- 第二项
- 第三项
```

任意一组的效果如下：

* 第一项
* 第二项
* 第三项

##### 有序列表

有序列表使用数字加`.`表示，`.`后空一格，源代码如下：

```markdown
1. 第一项
2. 第二项
3. 第三项
```

1. 第一项
2. 第二项
3. 第三项

##### 列表嵌套

在子列表的选项前添加两个或四个空格，嵌套列表可以是无序或有序列表。源代码如下：

```markdown
1. 第一项
    - 第一项嵌套的第一个元素
    - 第一项嵌套的第二个元素
2. 第二项
    1. 第二项嵌套的第一个元素
    2. 第二项嵌套的第二个元素
```

效果如下：

1. 第一项
    - 第一项嵌套的第一个元素
    - 第一项嵌套的第二个元素
2. 第二项
    1. 第二项嵌套的第一个元素
    2. 第二项嵌套的第二个元素

#### Markdown区块

在段落开头使用`>`符号，然后紧跟一个**空格**符号即可。效果如下：

> 示例文字
>
> 示例文字
>
> 示例文字

与列表类似，区块也可以嵌套，一个`>`是最外层，两个`>`是第一层嵌套，依此类推：

> 最外层
>
> > 第一层套娃
> >
> > > 第二层套娃

区块和列表可以结合起来使用，源代码及效果如下：

```markdown

> 1. 第一项
> 2. 第二项
> - 第一项
> + 第二项
> * 第三项

- 第一项
    > 列表中
    > 使用区块
- 第二项
```

> 区块中嵌套列表
> 1. 第一项
> 2. 第二项
> - 第一项
> + 第二项
>
> * 第三项

- 第一项
    > 列表中
    > 使用区块

#### Markdown链接

简单链接会在[其他常用命令](#其他常用命令)一节中介绍，这里介绍高级链接方式。源代码示例及效果如下所示：

```markdown
链接使用 1 作为文档变量[第三小节][1]

链接使用 2 作为网站变量[baidu][2]

文档结尾为变量赋值

[1]: #Markdown链接

[2]: https://www.baidu.com/
```

链接使用 1 作为文档变量[第三小节][1]

链接使用 2 作为网站变量[baidu][2]

文档结尾为变量赋值

[1]: #Markdown链接

[2]: https://www.baidu.com/

> 1. 可以看到使用这种方式实现文档内跳转美观程度不够，建议通过简单链接方式实现文档内跳转
> 2. 这种方式比较类似论文末尾的参考文献编号，但是格式规范及出现位置还需要调整，比较适合记录网络资源学习博客

#### Markdown表格

使用`|`分隔不同单元格，使用`-`分隔表头和其他行

对齐方式：

- `-:`设置内容和标题栏居右对齐
- `:-`设置内容和标题栏居左对齐
- `:-:`设置内容和标题栏居中对齐

源代码如下：

```markdown
|左对齐|右对齐|居中对齐|
|:----|----:|:----:|
|单元格|单元格|单元格|
|单元格|单元格|单元格|
```

效果如下：

| 左对齐 | 右对齐 | 居中对齐 |
| :----- | -----: | :------: |
| 单元格 | 单元格 |  单元格  |
| 单元格 | 单元格 |  单元格  |

Markdown原生表格不支持单元格合并操作，可以通过插入HTML中的table满足这一需求，关于HTML表格绘制可以自行上网查阅学习。下面会将其他常用命令以表格形式汇总。

> Typora中对表格工具进行优化，可以使用图形化界面对表格行列数、对齐方式进行设置
